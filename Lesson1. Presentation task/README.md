# ML Engineering   
*ITMO University × Karpov.Courses*

## Проект: Рекомендательная система докладов JUG.ru

## 1. Бизнес-анализ (Business Understanding)

### Бизнес-цель проекта

Создание **первой в России гибридной (онлайн + оффлайн) рекомендательной системы** для технических IT-конференций с фокусом на IML, SmartData и VideoTech.

#### Ключевые бизнес-цели:
- **Повысить удовлетворённость участников** за счёт персонализированных рекомендаций докладов.
- **Увеличить вовлечённость** через активный сбор обратной связи.
- **Оптимизировать заполняемость залов**.
- **Сформировать ценный датасет** пользовательских предпочтений для будущих рекомендаций.

На момент запуска проекта **на рынке отсутствуют специализированные решения**, учитывающие:
- **Семантический анализ контента докладов** (с использованием NLP),
- **Гибридную природу взаимодействия** (онлайн+оффлайн),
- **Активный сбор фидбека в процессе участия**.

Таким образом, проект является **первым в своём роде в российском IT-сообществе**.

## 1.1 Текущая ситуация (Assessing Current Solution)

### Ресурсы и данные
- **Данные**: архивы докладов (видео, слайды, текстовые описания), рейтинги зрителей, экспертные оценки программных комитетов, статистика просмотров.
- **Инфраструктура**: не требуется закупка оборудования. Для PoC и демо используется облачная среда; интеграция с Telegram — через Bot API.
- **Экспертная поддержка**:  
  **Марк Паненко**, Chief Data Scientist Ozon Fintech, член программных комитетов IML и SmartData — выступает личным наставником команды.

### Риски и меры по их снижению

| Риск | Мера снижения |
|------|----------------|
| **Срыв сроков** | • Чёткое распределение ролей в команде из 3–4 человек<br>• Еженедельные демо и ревью с наставником<br>• Использование готовых библиотек |
| **Проблема cold start** | • Синтетическая генерация пользовательских профилей на старте<br>• Активный сбор данных через Telegram-бота (вопросы при регистрации + оценки после докладов) |
| **Низкая предсказуемость предпочтений** | • Гибридный подход: семантический анализ контента + поведенческие сигналы<br>• При отсутствии истории — рекомендации на основе профиля (роль, стек, уровень) и популярности |

## 1.2 Решаемые задачи с точки зрения аналитики (Data Mining Goals)

### Целевые метрики качества
- **Precision@5** и **Recall@5** — основные метрики релевантности рекомендаций.
- **NDCG@5** — для оценки качества ранжирования.
- **User Satisfaction Rate**, **Mean Reciprocal Rank (MeanRR)** — для измерения субъективного восприятия рекомендаций (сбор через бота).

### Критерии успешности
- Минимальный и целевой пороги будут уточнены совместно с заказчиком на этапе постановки задачи.

### Возможные подходы к решению
- **Content-based filtering** — на основе NLP-эмбеддингов докладов.
- **Hybrid models** — комбинация контентных и коллаборативных сигналов.
- **Семантический вектор + пользовательский профиль** — для персонализации в условиях ограниченной истории.
